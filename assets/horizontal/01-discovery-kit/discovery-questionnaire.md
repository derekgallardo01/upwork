# Discovery questionnaire

Work through these on the first call (or send ahead). Skip nothing in **Access &
constraints** — that's where hidden blockers live.

## 1. Outcome
- In one sentence, what should be true when this is done?
- What does success look like in numbers (hours saved/week, errors avoided, report
  produced by X)?
- Who is the end user, and how technical are they?

## 2. Current state
- What's the process today, step by step? (Who does what, in which tool?)
- What's the trigger — schedule, an email, a form, a manual click?
- Where does the data start, and where does it need to end up?
- What have you already tried? What broke?

## 3. Systems & data
- Which platforms are involved? (M365 / SharePoint / Power Platform / Azure /
  third-party apps — names and versions if known.)
- Roughly how much data, how often? (rows/day, files/week)
- Any sensitive data (PII, financial, health)? Compliance regime?
- Sample data or a screenshot of the current spreadsheet/report?

## 4. Access & constraints
- Can you grant a test environment / sandbox tenant? Who controls licensing?
- Which connectors/APIs are licensed and available?
- Any "must-use" or "must-not-use" tools?
- Hard deadline or budget ceiling?

## 5. Scope & handover
- Is this a one-time build or ongoing?
- Who maintains it after — you, an internal team, or me?
- What level of documentation/training do you need?
- What's explicitly *out* of scope for this engagement?
